# 🔴 RED | NOTE

**RED-NOTES** is a comprehensive, structured knowledge base and field manual for Offensive Security, built to consolidate practical skills, methodologies, and real-world tradecraft across the full red teaming lifecycle.

This project is designed to serve as both a **learning platform** and an **operational reference**, covering everything from foundational concepts to advanced adversary simulation techniques.

---

## 🧠 Overview

RED-NOTES represents an end-to-end offensive security framework, documenting the tools, tactics, and procedures (TTPs) used in modern red teaming, penetration testing, and adversary emulation.

It is organized to reflect how real operations unfold — from reconnaissance to post-exploitation — while also diving deep into specialized areas such as evasion, malware development, and AI-assisted operations.

---

## 🧱 Project Philosophy

RED-NOTES is built on a few core principles:

* **Practical over theoretical** – Focus on techniques that work in real environments
* **Operator mindset** – Think like an adversary, not just a tester
* **Depth over breadth** – Understand *why* techniques work, not just *how*
* **Continuous evolution** – Offensive security is always changing, and so is this project

---

## 📁 Structure

The repository is organized into modular sections, each representing a phase of the offensive lifecycle or a specialized discipline. Each section contains:

* Notes and explanations
* Step-by-step methodologies
* Tooling references
* Real-world techniques and edge cases

---

## 🚀 Goals

* Build a **personal offensive security encyclopedia**
* Serve as a **red team field manual**
* Document **advanced tradecraft and methodologies**
* Enable **end-to-end adversary simulation**
* Continuously refine skills in:

  * Active Directory exploitation
  * Evasion & OPSEC
  * Phishing & social engineering
  * Malware development
